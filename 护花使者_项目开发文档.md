## 零、开发约束（重要）

### 0.1 成本原则
**本项目软件部分必须 0 成本运行**，所有第三方服务均采用免费方案，不引入任何付费订阅或按量计费服务。

### 0.2 数据存储策略
**当前阶段所有数据纯本地存储**，不依赖服务器：
- 用户预设 → 本地 SQLite (Drift)
- 作业历史/飞行记录 → 本地 SQLite (Drift)
- 设备配对信息 → 本地 SharedPreferences
- 小课堂收藏/浏览历史 → 本地 SQLite (Drift)
- 用户偏好设置 → 本地 SharedPreferences

**注意**：APP 卸载后本地数据会丢失。后续版本增加云端同步功能后可解决此问题。

### 0.3 天气数据方案（0成本）
采用 **Open-Meteo 开源天气 API**：
- **完全免费**，无需 API Key，无需注册，无调用次数限制
- 数据来源：ECMWF、NOAA、DWD 等 30+ 国家气象机构官方模型
- 全球覆盖，最高 1km 分辨率
- 提供字段：温度、风速风向、湿度、降雨概率、气压等植保所需全部数据
- 平均响应 < 10ms
- 离线降级：无网络时显示"无法获取天气"提示，不影响其他功能

## 一、项目概述

### 1.1 项目定位
**护花使者** 是一款面向农业植保领域的智能无人机操控 APP，通过 AI 技术将传统复杂的植保无人机作业流程简化为"一键式"操作，降低使用门槛，提升作业效率。项目包含三大平台：**手机APP端**、**无人机飞控平台**、**遥控器中继平台**。

### 1.2 核心价值
- 从"专业飞手依赖"到"农户自主操作"的跨越
- AI 驱动的作业参数自动决策，告别手动调参
- 离线/在线双模式，无网络环境也能正常作业
- 科学种植知识库，授人以渔

### 1.3 对标产品参考
- 大疆农服（DJI Ag Platform）：全流程作业管理、地块管理、大疆学堂
- 极飞农服（XAG Farm Service）：AI农事决策、数字地块建模、团队协同
- 飞防管家：中小团队友好、实时监控、药剂库管理

## 二、APP端开发方案

### 2.1 技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 跨平台框架 | **Flutter 3.x** | 一套代码覆盖 iOS + Android + 鸿蒙等，渲染性能接近原生，社区活跃 |
| 状态管理 | **Riverpod 2.x** | 编译时安全、无上下文依赖、适合复杂业务场景 |
| 地图引擎 | **高德地图/腾讯地图 Flutter SDK** | 地块圈选、航点打点、面积计算、离线地图支持 |

| 网络请求 | **Dio + Retrofit** | HTTP/HTTPS 请求，拦截器统一处理鉴权/加密 |

| AI 集成 | **云端 API + 端侧 TFLite** | 核心AI推断走云端，离线场景本地轻量模型降级 |
| 推送通知 | **Firebase Cloud Messaging / 极光推送** | 天气预警、作业完成通知、系统消息 |

### 2.5 地图圈选功能详细设计

这是整个APP交互最复杂的模块，需要精细设计：

```
地图界面布局：
┌─────────────────────────────────┐
│ 顶部状态栏（已添加航点数：N）      │
├─────────────────────────────────┤
│                                 │
│                                 │
│             +  ← 十字准心(固定)   │
│                                 │
│                                 │
├─────────────────────────────────┤
│ [撤销]  [添加航点/闭合区域]  [完成] │
│ 底部按钮区                       │
└─────────────────────────────────┘

交互规则：
1. 十字准心固定在地图中央，用户滑动地图来定位
2. 点击"添加航点" → 在十字准心对应位置添加红色圆点
3. 添加第2个航点 → 自动用红色细线连接两点
4. 添加第3个航点 → 第2和第3个航点自动连线，以此类推
5. 当十字准心对准第1个航点：
   - "添加航点"按钮变为"闭合区域"按钮
   - 点击闭合区域完成圈选
6. 点击"撤销" → 移除最后一个航点，连线同步消失
   - 撤销到0个航点时，撤销按钮消失
7. 点击"完成" → 
   - 自动计算面积（使用球面多边形面积算法）
   - 面积 < 最小作业范围 → 弹窗提示"该区域未达到无人机的最小作业范围"
   - 面积符合条件 → 区域呈现透明浅蓝色填充效果 + 自动规划航路
8. 支持手势：双指缩放、单指滑动
9. 支持地图类型切换：卫星图/标准地图
10. 支持离线地图：提前下载作业区域离线地图包
```

**技术实现要点：**
- 使用 `Stack` 叠加层实现十字准心
- 使用 `CustomPainter` 绘制航点和连线
- 使用 `Polygon` 绘制闭合区域半透明填充
- 球面多边形面积计算（考虑地球曲率）
- 航路自动规划算法（平行往复式航线，考虑喷幅重叠率）

### 2.6 通信协议设计

#### 2.6.1 手机 ↔ 遥控器 蓝牙通信协议

采用自定义二进制协议帧，格式如下：

```
协议帧结构（最大256字节）：
┌──────┬──────┬──────┬──────┬──────────┬──────┬──────┐
│ 帧头  │ 版本  │ 命令  │ 长度  │  数据段   │ 校验  │ 帧尾  │
│ 2B   │ 1B   │ 1B   │ 2B   │  N Byte  │ 2B   │ 2B   │
│0xAA55│ 0x01 │ CMD  │ LEN  │  DATA    │ CRC16│0x55AA│
└──────┴──────┴──────┴──────┴──────────┴──────┴──────┘
```

**命令类型（CMD）：**

| 命令码 | 名称 | 方向 | 说明 |
|--------|------|------|------|
| 0x01 | 设备握手 | APP→RC | 建立连接，交换设备信息 |
| 0x02 | 握手应答 | RC→APP | 确认连接，返回设备信息 |
| 0x10 | 参数下发 | APP→RC | 飞行参数、航路数据、喷洒参数 |
| 0x11 | 参数确认 | RC→APP | 参数接收确认/错误重传请求 |
| 0x20 | 任务控制 | APP→RC | 起飞/暂停/继续/返航/降落 |
| 0x21 | 任务状态 | RC→APP | 实时飞行状态回传 |
| 0x30 | 心跳包 | 双向 | 1Hz心跳，维持连接 |
| 0x40 | 固件升级 | APP→RC | 遥控器固件升级 |
| 0xFF | 错误报告 | RC→APP | 异常状态上报 |

**数据段示例（参数下发 0x10）：**
```
飞行高度:    2B (uint16, 单位: dm, 范围: 10-100)
飞行速度:    2B (uint16, 单位: cm/s, 范围: 100-1500)
喷洒量:      2B (uint16, 单位: ml/亩, 范围: 100-10000)
喷幅:        2B (uint16, 单位: cm, 范围: 100-1000)
风速修正角:   1B (int8, 单位: 度, 范围: -30~+30)
航点数:      1B (uint8, 范围: 3-50)
航点列表:    N*8B (每个航点: lat 4B + lng 4B, float32)
作业开关:     1B (0x00关/0x01开)
```

#### 2.6.2 遥控器 ↔ 无人机 数传通信

遥控器与无人机之间采用 2.4GHz/915MHz 无线数传模块通信，实现远距离（≥3km）高可靠数据链路。

**通信参数：**
- 频率：2.4GHz ISM 或 915MHz（根据不同地区法规）
- 功率：可调，最大 1W（满足农田大范围作业）
- 速率：≥ 115200bps
- 协议：MAVLink 精简版 / 自定义协议
- 延迟：< 50ms

**遥控器下发数据：**
- 飞行参数（高度、速度、喷洒参数）
- 航路航点列表
- 飞行控制指令（起飞/降落/返航/暂停）
- 喷洒开关控制

**无人机回传数据：**
- 实时姿态（俯仰/横滚/偏航）
- GPS 位置（经纬度、高度）
- 电量/电压
- 飞行速度、飞行模式
- 喷洒状态（流速、剩余药量）
- 传感器状态（IMU、气压计、磁力计）
- 异常告警（低电量、GPS丢失、通信中断）

Phase 4：遥控器开发
  ├── 硬件：STM32主控 + 2.4G数传 + 蓝牙模块 + 显示屏 + 摇杆

  └── 量产工艺文件
